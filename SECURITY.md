# Security Policy

## Supported versions

This project is a template/starter repository. Security updates are applied to
the latest `main` branch state.

## Reporting a vulnerability

Please do not open public issues for suspected vulnerabilities.

Instead, report privately to:

- maintainer@example.com

Include:
- A clear description of the issue
- Reproduction steps or proof of concept
- Potential impact
- Suggested mitigation (if available)

## Disclosure process

1. We acknowledge receipt within 3 business days.
2. We validate and triage the report.
3. We prepare a fix and release guidance.
4. We coordinate disclosure timing with the reporter.
